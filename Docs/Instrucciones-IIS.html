Guía de Despliegue para IIS (URL WPT)

Requisito Fundamental: Asegúrate de que tu servidor IIS tiene instalado el .NET Core 8 Hosting Bundle. Si no lo tienes, el servidor IIS devolverá un error HTTP 500.19 en el Backend.

🌍 Fase 1: Configurar el Backend (Servicios DGII)

La API que se comunica con la DGII y expone los servicios web.

  • Abre el Administrador de Internet Information Services (Ejecutar > inetmgr).
  • Haz clic derecho en Grupos de aplicaciones (App Pools) y selecciona Agregar grupo de aplicaciones.
    • Nombre: WptApiPool
    • Versión de .NET CLR: Sin código administrado (No Managed Code). ¡Esto es vital para .NET Core 8!
  • Ve a Sitios, haz clic derecho y selecciona Agregar sitio web...
  • Aplica la siguiente configuración:
    • Nombre del sitio: WPT_Servicios_DGII
    • Grupo de aplicaciones: Pólmalo a WptApiPool
    • Ruta de acceso física: e:\Empresas\GMV\Proyectos Antigravity\URL WPT\Publish_API
    • Puerto (Binding): Asígnale el puerto / dominio de despliegue principal.

💻 Fase 2: Configurar el Frontend (Manager Web)

El portal web de pruebas (Aplicativo Frontend).

  • Volviendo al apartado de Sitios en IIS, haz clic derecho y selecciona Agregar sitio web...
  • Aplica la siguiente configuración:
    • Nombre del sitio: WPT_Manager_Web
    • Grupo de aplicaciones: Puedes elegir el DefaultAppPool (No afecta a los estáticos de React).
    • Ruta de acceso física: e:\Empresas\GMV\Proyectos Antigravity\URL WPT\Publish_Web
    • Puerto (Binding): Asígnale un puerto distinto (Por ejemplo el puerto 80) o subdominio particular.
